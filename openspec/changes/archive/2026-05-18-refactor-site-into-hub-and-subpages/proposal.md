## Why

当前站点是单页静态页面，首页直接承载“你画我猜”游戏逻辑、词库数据和展示内容，缺少主站入口、统一导航和独立子页结构。这使得后续新增页面、扩展小游戏或沉淀主站内容时维护成本高，也无法为每个子页面提供稳定独立的访问链接。

本次重构的目标是将站点调整为“主站首页 + 独立子页面”的信息架构，把现有“你画我猜”迁移为一个独立子页，并在主站上以卡片形式统一展示可访问的子页面，为后续继续扩展更多内容页或互动页建立可复用的静态站基础。

## What Changes

- 将现有单页站点重构为“主站首页 + 子页面”结构。
- 将现有“你画我猜”从首页迁移到独立子页面，并保留其现有核心玩法。
- 在主站首页增加子页面卡片展示区，按预设顺序展示站内各个子页面入口。
- 为每个子页面提供独立、稳定、可直接访问的网页链接。
- 抽离主站通用布局、页面元信息和静态资源组织方式，建立便于后续新增页面的目录约定。
- 将“你画我猜”的页面逻辑与数据资源从首页中解耦，避免继续依赖单一页面中的全局脚本堆叠。
- 允许后续新增子页面时仅需增加页面目录与卡片配置，无需重写主站整体结构。

## Capabilities

### New Capabilities
- `site-home-hub`: 定义主站首页作为统一入口页，展示站点简介和按顺序排列的子页面卡片列表。
- `subpage-routing`: 定义子页面的独立目录结构与访问链接规则，确保每个子页面可单独访问。
- `draw-and-guess-page`: 定义“你画我猜”作为独立子页面存在，并保留其词条随机展示和单次选择锁定等核心交互。
- `site-page-scaffold`: 定义主站与子页面共享的页面骨架、资源组织约定和扩展方式，支撑后续新增页面。

### Modified Capabilities
- None.

## Impact

- Affected code:
  - `index.html`
  - `config.js`
  - 现有静态资源引用方式
  - 新增主站/子页面目录与公共资源目录
- Affected systems:
  - 站点信息架构
  - 页面入口与链接组织方式
  - 前端静态资源加载关系
- Dependencies:
  - 可能保留部分现有 CSS/JS 资源，也可能在设计阶段裁剪旧主题依赖
- APIs:
  - 无后端 API 变更
- Breaking impact:
  - 首页不再直接承载“你画我猜”游戏内容，原游戏入口将迁移到独立子页面
