## ADDED Requirements

### Requirement: COA 控制台作为独立子页面提供
系统 SHALL 将 COA 控制台作为主站中的独立子页面提供，并使用与现有作品集子页面一致的共享页面骨架、返回首页入口和独立可访问链接。

#### Scenario: 访客从首页进入 COA 控制台
- **WHEN** 访客打开主站首页并查看项目列表
- **THEN** 系统展示一个指向 COA 控制台子页面的卡片入口
- **THEN** 访客可以通过该入口进入独立的 COA 控制台页面

#### Scenario: 访客直接访问 COA 控制台子页面
- **WHEN** 访客直接访问 `/pages/coa-controller/`
- **THEN** 系统直接加载 COA 控制台页面而不依赖首页先执行
- **THEN** 页面展示清晰的返回首页入口与项目说明

### Requirement: COA 控制台页面保持统一站点风格
系统 SHALL 让 COA 控制台页面复用主站公共视觉语言，包括极简排版、统一面板样式、克制的按钮与表单外观，并仅用有限的语义状态色表达成功、警告和错误。

#### Scenario: 访客查看 COA 控制台页面
- **WHEN** 访客进入 COA 控制台子页面
- **THEN** 页面整体布局、标题层级、面板样式和导航方式与现有作品集页面保持一致
- **THEN** 页面不会继续呈现一套明显脱离主站的独立旧视觉

### Requirement: Gist 连接信息由本地输入配置
系统 SHALL 要求用户在页面中输入 GitHub Token 与 Gist ID 作为连接配置，并将这两个值仅缓存到当前浏览器本地，而不是写回远端 Gist 文件。

#### Scenario: 页面首次打开且本地没有配置
- **WHEN** 用户首次打开 COA 控制台页面且浏览器中没有缓存 Token 或 Gist ID
- **THEN** 页面提示用户先输入必要的连接配置
- **THEN** 页面不会在缺少配置时继续发起 Gist 读取或写入

#### Scenario: 用户再次打开页面
- **WHEN** 用户此前已经在当前浏览器输入过 Token 与 Gist ID 并刷新或重新打开页面
- **THEN** 页面自动从本地缓存回填这两个输入项
- **THEN** 页面仍然将其视为本地连接配置而非远端状态数据

### Requirement: 控制台核心交互继续面向所配置的 Gist 生效
系统 SHALL 基于用户当前输入的 Gist ID 继续支持状态刷新、固定动作发送、账号别名缓存、单人囚笼刷币选项和频率限制等既有控制台核心行为。

#### Scenario: 用户刷新状态
- **WHEN** 用户已输入有效的 Token 与 Gist ID 并执行刷新状态
- **THEN** 页面从该 Gist 的 `status.json` 读取状态并更新状态卡和结果摘要
- **THEN** 页面继续应用刷新冷却限制和错误提示

#### Scenario: 用户发送固定动作
- **WHEN** 用户已输入必要配置并点击启动或停止
- **THEN** 页面向当前 Gist 的 `command.json` 写入对应动作载荷
- **THEN** 页面继续应用账号别名校验、单人囚笼刷币参数拼装和操作冷却限制
